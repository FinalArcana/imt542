## **Project Overview**

This project integrates two USDA datasets: **Foundation Foods** and **F‑MAP (Food‑at‑Home Monthly Area Prices)**, to evaluate the nutritional value of foods relative to their cost. The system ingests both datasets, normalizes their units, and produces a unified JSON structure that supports downstream analysis such as protein‑per‑dollar and calorie‑per‑dollar comparisons.

The system operates in three stages:

1. **Ingest Foundation Foods nutrient data and F‑MAP price data**  
2. **Normalize units and align both datasets to a shared 100‑gram basis**  
3. **Produce a structured JSON output with explicit units, provenance metadata, and derived metrics**

To keep the structure self‑describing without increasing payload size, the improved JSON includes lightweight metadata linking to the official documentation for each dataset.

---

# **1. How Our Structure Promotes Maintaining and Measuring Data Quality**

The redesigned structure improves data quality by making units explicit, aligning measurement bases, and adding minimal provenance metadata.

### **A. Explicit Value + Unit Scheme Improves Clarity**

Foundation Foods expresses nutrients as structured objects:

```json
"nutrients": {
  "protein": { "value": 21.2, "unit": "g" },
  "energy": { "value": 120, "unit": "kcal" }
}
```

Keeping this scheme prevents unit ambiguity and supports automated validation (e.g., ensuring protein is always in grams).

### **B. Shared 100‑Gram Basis Ensures Consistent Comparisons**

Foundation Foods uses a 100‑gram basis, while F‑MAP reports price per pound.  
The improved structure converts F‑MAP prices to:

```json
"price": { "value": 0.646, "unit": "USD_per_100g" }
```

This alignment eliminates conversion errors and ensures nutrients and prices are directly comparable.

### **C. Lightweight Metadata Supports Transparency**

A compact `datasetInfo` block links to the official USDA sources:

```json
"datasetInfo": {
  "foundationFoods": { "source": "https://fdc.nal.usda.gov/" },
  "fmap": { "source": "https://www.ers.usda.gov/data-products/food-price-outlook/" }
}
```

This keeps the payload small while making the data sources discoverable and auditable.

### **D. Normalized Identifiers Improve Traceability**

Storing both `fdc_id` and `efpg_code` ensures consistent mapping between nutrient and price data and reduces mismatches.

# **2. How Our Structure Enables Identification and Management of Information Security Issues**

Although the USDA datasets used in this project are public, the improved structure incorporates several design choices that help identify and manage information security issues related to data integrity and pipeline reliability.

### **A. Status Codes and Timestamps Detect Ingestion Anomalies**

Each dataset fetch includes a `status` code and `lastFetched` timestamp:

```json
"sourceFetch": {
  "foundationFoods": { "lastFetched": "...", "status": 200 },
  "fmap": { "lastFetched": "...", "status": 200 }
}
```

This allows the system to detect incomplete downloads, unexpected update gaps, or upstream anomalies that may indicate tampering or corruption.

### **B. Schema Versioning Prevents Unauthorized or Accidental Changes**

The structure includes:

```json
"schemaVersion": "1.0.0"
```

This enables validation against a known schema and makes structural drift detectable before it affects downstream analysis.

### **C. Source Links Support Authenticity Verification**

By storing the official USDA URLs in `datasetInfo`, the system can verify that data originated from the correct domains. This reduces the risk of ingesting spoofed or unofficial datasets.

### **D. Consistent Units and Basis Amounts Prevent Manipulation**

The structure enforces:

- explicit units for every nutrient  
- a fixed `basis_amount_g` of 100 g  
- a normalized price unit (`USD_per_100g`)  

This prevents silent manipulation such as switching kcal ↔ kJ or inserting per‑serving values.

### **E. Separation of Raw and Derived Metrics Supports Integrity Checks**

The structure stores raw nutrient and price data separately from derived metrics:

```json
"derivedMetrics": {
  "protein_per_dollar": { "value": 32.8, "unit": "g_per_USD" }
}
```

This allows the system to recompute derived values and verify that they match the stored results, helping detect corruption or unauthorized modification.
